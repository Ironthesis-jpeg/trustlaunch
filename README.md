TrustLaunch

Tagline

Build trust before hype.

---

Vision

TrustLaunch is a Base-native token launch platform focused on transparency, fairness, and founder accountability.

Instead of asking investors to trust founders blindly, TrustLaunch provides public trust metrics, founder reputation, and transparent launch standards.

---

Core Principles

- Fixed Supply
- No Mint Function
- Founder Allocation = 0%
- Team Allocation = 0%
- LP Burned Forever
- Creator Fee Transparency
- Founder Reputation
- Trust Before Hype

Token Rules

Every token launched through TrustLaunch must follow:

- Fixed Supply
- No Minting
- No Hidden Allocations
- No Team Allocation
- No Founder Allocation
- LP Burned Forever

---

Creator Fee

Allowed Values:

- 0%
- 1%
- 2%

Creator Fee applies only to:

- Buys
- Sells

Creator Fee does NOT apply to:

- Transfers
- Airdrops
- Wallet-to-Wallet Transactions

Creator Fee is collected in tokens and sent to the founder wallet.

Treasury Wallet

Base Address:

0x18b0eE032e93918254B9336c1aB24AC9a336c7F2

Treasury receives liquidity contributions according to launch type.

Treasury does not receive token allocations.

---

Liquidity Model

All token supply goes into liquidity.

LP Tokens are burned forever.

Liquidity can never be withdrawn.

---

Meme Escrow Model

If a Meme launch chooses "No Liquidity":

- 100% token supply is sent to TrustLaunch Escrow.
- Founder receives 0 tokens.
- Token is not tradable.
- Founder can add liquidity later.
- Once liquidity is added:
  - Pool is created.
  - LP is burned forever.
  - Trading becomes active.

Governance

TrustLaunch uses an admin-controlled model for platform settings.

Admin may update:

- Launch Fees
- Treasury Wallet
- Featured Listing Rules

Admin may NOT change:

- Founder Allocation Rules
- Team Allocation Rules
- LP Burn Rules
- Creator Fee Cap
- Token Supply Rules
